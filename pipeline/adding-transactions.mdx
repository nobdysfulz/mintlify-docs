# Adding and Editing Transactions

## Adding a New Transaction

Click **Add Transaction** from the Pipeline page. A form opens with all the fields for the transaction.

### Transaction Fields

| Field | Description |
|-------|-------------|
| **Address** | Full property address |
| **Client Name** | Buyer or seller name |
| **Type** | Listing or Buyer |
| **Status** | Starting status (usually Pre-Signed or Active) |
| **List Date** | Date the listing went live or the buyer agreement was signed |
| **Expiry Date** | For listings: when the listing agreement expires |
| **Close Date** | Expected or actual closing date |
| **Price** | Sale price or list price |
| **GCI** | Gross commission income for this transaction |
| **Lead Source** | Where this client came from (used in ROI Tracker) |
| **Notes** | Any additional context about the deal |

## Editing a Transaction

Click any transaction row in the pipeline list to open the detail view. Update any field and save. Status changes take effect immediately and are reflected in the Dashboard, Leaderboard, and all reporting.

## Updating Status as the Deal Moves

1. Pre-Signed when you are working toward a signed agreement
2. Active when the listing goes live or the buyer starts searching
3. Pending when you go under contract
4. Closed when the transaction closes

If a deal falls apart, mark it x-Cancelled or y-Sale Failed so it stops appearing as active.

## Adding GCI

GCI should reflect the commission your side earns. For listings, enter your side's commission. For buyer deals, enter your side's commission.

## Deleting a Transaction

If a transaction was entered in error, open the detail view and use the delete option. If the deal fell through but you want to keep the record, use x-Cancelled or y-Sale Failed instead.
